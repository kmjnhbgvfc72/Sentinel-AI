# SOC Response Workflow

1. Phase 4 emits a prediction and risk score.
2. Phase 5 maps the event to relationships and possible paths.
3. Analysts review path severity and affected assets.
4. The recommendation engine proposes account review, telemetry preservation, approved patching, indicator validation, and monitoring.
5. SOC personnel decide and execute defensive actions under existing authorization and change-control procedures. Phase 5 does not isolate hosts, modify access, exploit vulnerabilities, or execute commands automatically.
